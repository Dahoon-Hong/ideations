# **게임 기획안: Chroma Seek (가제)**

## **1\. 개요 (Overview)**

* **장르:** 퍼즐 / 아케이드  
* **플랫폼:** Web / Mobile  
* **타겟 유저:** 빠른 판단력을 즐기는 캐주얼 게이머, 색감 능력을 테스트하고 싶은 유저  
* **핵심 재미:** 미묘한 색의 차이를 빠르게 인지하고 반응하는 쾌감과 긴장감

## **2\. 게임 규칙 (Core Mechanics)**

### **2.1. 기본 플레이**

1. 화면에 **$N \\times N$** 격자(Grid) 형태의 타일들이 생성된다.  
2. 모든 타일은 동일한 기본 색상(Base Color)을 가진다.  
3. 그중 **$T$개**의 타일(Target Tile)만이 미세하게 다른 색상을 가진다.  
4. 유저는 제한 시간 내에 다른 색상의 타일을 모두 찾아 클릭(터치)해야 한다.  
5. 정답을 맞히면 다음 스테이지로 넘어가거나 점수를 획득하며, 오답 시 페널티가 주어진다.

### **2.2. 색상 생성 알고리즘**

* **기본 색상(Base Color):** $RGB(R, G, B)$는 0\~255 사이에서 랜덤 생성.  
* **정답 색상(Target Color):** $R, G, B$ 중 하나의 채널을 무작위로 선택하여 난이도 값($\\Delta$)만큼 더하거나 뺀다.  
  * 예: $\\Delta \= 50$, 선택 채널 $R$ $\\rightarrow$ 정답 색상은 $(R+50, G, B)$  
  * 단, 0\~255 범위를 벗어나지 않도록 보정(Clamping)한다.

## **3\. 레벨 디자인 (Level Design)**

총 9개의 스테이지(Stage)로 구성되며, 3가지 변수가 복합적으로 작용하여 난이도를 조절한다.

### **3.1. 난이도 변수**

1. **Grid Size ($N$):** 격자의 크기 ($7 \\times 7$ \~ $9 \\times 9$)  
2. **Target Count ($T$):** 찾아야 하는 타일의 개수 (1 \~ 3개)  
3. **Color Delta ($\\Delta$):** 색상 차이의 정도 (100 \~ 10, 숫자가 작을수록 어려움)

### **3.2. 스테이지별 구성표**

| Stage | Grid (N×N) | Target Tiles (T) | Color Delta (Δ) | 난이도 설명 |
| :---- | :---- | :---- | :---- | :---- |
| **1** | $7 \\times 7$ | 1 | 100 | **입문:** 색 차이가 확연하고 타겟이 하나임 |
| **2** | $8 \\times 8$ | 1 | 90 | 타일이 조금 많아지고 색 차이가 줄어듦 |
| **3** | $9 \\times 9$ | 1 | 80 | 타일이 많아 찾는 시간이 조금 더 걸림 |
| **4** | $7 \\times 7$ | 2 | 70 | **중급:** 그리드는 작아졌으나 찾아야 할 타겟이 2개로 증가 |
| **5** | $8 \\times 8$ | 2 | 60 | 타겟 2개를 유지하며 색 차이가 좁혀짐 |
| **6** | $9 \\times 9$ | 2 | 50 | 넓은 그리드에서 2개를 찾아야 함 |
| **7** | $7 \\times 7$ | 3 | 40 | **고급:** 타겟이 3개로 증가, 높은 집중력 요구 |
| **8** | $8 \\times 8$ | 3 | 25 | 색 차이가 육안으로 구별하기 힘들어짐 |
| **9** | $9 \\times 9$ | 3 | 10 | **최고 난이도:** 거의 비슷한 색상, 넓은 맵, 많은 타겟 |

* *Note: Color Delta는 스테이지 진행에 따라 100에서 10까지 선형적 혹은 계단식으로 감소하도록 설계함.*

## **4\. 점수 및 시스템 (Scoring & System)**

### **4.1. 점수 산정 방식 (Score Formula)**

점수는 \*\*속도(Speed)\*\*와 \*\*정확도(Accuracy)\*\*를 기반으로 계산한다.

$$Score \= (Base Score \\times \\Delta\\\_Multiplier) \+ Time Bonus \- Penalty$$

* **Base Score:** 스테이지 클리어 기본 점수 (예: 1000점)  
* **$\\Delta\\\_Multiplier$:** 난이도(색상 차이)에 따른 가중치. (어려울수록 점수가 높음)  
* **Time Bonus:** (제한 시간 \- 소요 시간) $\\times$ 100  
* **Penalty:** 오답 클릭 횟수 $\\times$ 500점 차감 (혹은 시간 3초 차감)

### **4.2. Score Board**

* 게임 종료(9단계 클리어 혹은 타임오버) 시 최종 점수를 기록한다.  
* Local Storage를 활용하여 기기 내 최고 기록 Top 5를 표시한다.  
* **랭킹 표시 항목:** 순위, 점수, 도달한 스테이지

## **5\. UI/UX 구성**

### **5.1. 메인 화면 (Main Menu)**

* 게임 타이틀 로고  
* "Game Start" 버튼  
* "High Scores" (점수판) 버튼

### **5.2. 플레이 화면 (In-Game)**

* **상단 정보창:**  
  * 현재 Stage (예: 5 / 9\)  
  * 남은 목표 개수 (예: ● ○ / 2개 찾음)  
  * 현재 점수 및 남은 시간(타이머 바)  
* **중앙 영역:**  
  * $N \\times N$ 타일 그리드 컨테이너  
  * 반응형 디자인으로 모바일/PC 모두 정사각형 비율 유지  
* **피드백 효과:**  
  * **정답:** 경쾌한 효과음 \+ 해당 타일이 밝게 빛남 \+ 체크 표시  
  * **오답:** '버즈(Buzz)' 효과음 \+ 화면 흔들림(Shake) 효과

### **5.3. 결과 화면 (Result)**

* Total Score 표시  
* 성공 여부 (Clear / Game Over)  
* "Retry" 및 "Main Menu" 버튼
